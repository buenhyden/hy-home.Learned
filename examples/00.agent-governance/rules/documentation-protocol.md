# Documentation Protocol (March 2026)

This protocol defines the mandatory standards for project-wide documentation and README synchronization for AI Agents.

## 1. Mandatory Template Usage

All directory-level `README.md` files MUST be generated or updated using the **SSoT Template**:

- **Template Path**: `docs/99.templates/readme.template.md`

### Selection Guide

- **Implementation Layer** (`web/`, `server/`, `app/`, etc.): Use `Base Structure` + `Implementation Snippet` + `AI Agent Guidance Snippet`.
- **Ops & Utils** (`scripts/`, `docs/08.operations/`): Use `Base Structure` + `Ops & Utils Snippet`.

- **Docs & Governance**: Use `Base Structure` + `Docs & Governance Snippet`.

## 2. Synchronization Rules

Agents MUST synchronize documentation in the following scenarios:

1. **New Implementation**: When creating a new service or component directory, a `README.md` MUST be initialized immediately.
2. **Post-Task Completion**: After completing any implementation task (`Stage 06`), the agent MUST audit the local `README.md` and update it if the scope, scripts, or configuration have changed.
3. **Consistency Check**: During the final verification (`Stage 11`), the agent MUST verify that the `README.md` accurately reflects the state of the implementation.

## 3. Language Protocol

- **Header & Structure**: English
- **Description & Content**: Korean (Follow the project-wide "Korean for Humans" policy)
- **Technical Metadata**: English

## 4. Metadata Expectations

Every README generated by an agent MUST include the `AI Agent Guidance` snippet to ensure future agents understand the local guardrails and allowed outputs.
