# Git Workflow Rules

## Core Rules

- Never revert unrelated user changes.
- Do not use destructive history-rewriting commands unless explicitly requested.
- Do not amend commits unless explicitly requested.
- Validate the relevant checks after changes, including documentation-only changes when governance requires it.

## Documentation Changes

- If structure or responsibilities change, update the affected README files in the same workstream.
- Do not leave the repository in a state where root entrypoints and README navigation disagree.

## Review Readiness

- Keep changes scoped and explainable.
- Preserve clear traceability to PRD, Spec, and Plan documents.
