# Implementation Plan Standards

> **layer:** agentic
> **Status**: Active Detail Layer
> **Source**: Refactored from `.agent/rules/0100-Standards/0102-implementation-plan-standard.md`

## 1. Principles

- **Modular Authority**: Planning standards are partitioned into focused, composable rules.
- **Layering Precedence**: Regional or task-specific planning rules MUST NOT override these core standards.

## 2. Mandatory Reference Hierarchy

All implementation planning MUST comply with:

- Core Principles
- Task Specification & Validation
- Structured Autonomy Workflow
- Traceability & Verification
- Estimation & Work Hierarchy
- Templates

## 3. Procedures

- IF a conflict arises between sub-rules THEN refer to the Core Principles for resolution.

## 4. Validation Criteria

- [ ] All mandatory sub-rule links are valid and active.
- [ ] No unique technical planning concept is left unaddressed across the rule set.
