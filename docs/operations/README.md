# Operations

This directory stores operational records such as incidents and postmortems.

## What Belongs Here

- Active or recently resolved incident records
- Blameless postmortems
- Operational evidence that should remain linked from Plans or Runbooks

## Lazy Loading

1. Start from this index.
2. Open only the incident or postmortem needed for the active task.
3. Follow links to runbooks, plans, and specs only when they are relevant.
