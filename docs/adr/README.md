# Architecture Decision Records (ADR)

This directory stores decision records that explain why a significant
architectural choice was made.

## When To Use This Directory

- A new architectural boundary or platform rule is introduced
- An existing decision is superseded or deprecated
- A decision needs long-term traceability beyond a transient plan

## Lazy Loading

1. Read this index first.
2. Open only the ADR that matches the active decision or domain.
3. Follow links from that ADR to the related PRD, ARD, Spec, or Plan as needed.
