# Product Requirements Documents (PRD)

This directory stores the product and feature requirement documents for the
repository.

## Role In The Documentation Chain

- This is where requirement intent starts.
- Related decisions live in [ADRs](../adr/README.md).
- Related structure lives in [ARDs](../ard/README.md).
- Downstream implementation contracts live in [Specs](../specs/README.md).
- Governing rule:
  [0120-requirements-and-specifications-standard.md](../../.agent/rules/0100-Standards/0120-requirements-and-specifications-standard.md)

## Lazy Loading

1. Read this index first.
2. Open only the PRD for the current feature, domain, or platform concern.
3. Use linked ARDs, ADRs, Specs, and Plans only when they are required for the
   active task.

## Authoring Rules

- Use the [PRD Template](../../templates/prd-template.md).
- Keep the document focused on the "what" and "why".
- Define measurable success criteria and explicit boundaries.
- Link to the downstream Spec and Plan once they exist.
