# Repository Structure

This repository is organized to separate different types of knowledge and configurations.

### `TIL/` (Today I Learned)

Daily records of small technical learnings.

- **Structure**: `Year/Month/Day/Title.md`
- **Example**: `2026/01-Jan/13-python-uv-setup.md`

### `References/` (Knowledge Base)

Archives of external resources like books, lectures, and articles.

- **`Books/`**: Summaries and notes from technical books.
- **`Lecture-Data/`**: Lecture notes and resources.

### `.agent/` (AI Agents)

Configuration and rules for Agentic Workflows.

- **`rules/`**: Context rules for specific domains (Frontend, Backend, etc.).
- **`workflows/`**: Defined workflows for AI agents to follow.

### `.github/` (Project Management)

- **`ISSUE_TEMPLATE/`**: Templates for bug reports, feature requests, etc.
- **`instructions/`**: Detailed instructions for AI agents.
- **`agents/`**: Definitions of agent personas.
